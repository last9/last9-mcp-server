Replace an alert rule's evaluation configuration. First use get_alert_config or get_entity_alert_rules to retrieve the rule and verify entity_id. Pass the complete typed configuration in alert_rule; primary_indicator and expression_args are required, and partial evaluation updates are unsafe. For static rules, condition, alert_condition, and eval_window are required; eval_window is 0-60 minutes. Clients cannot set algorithm directly. Preserve properties and grouping settings when they should not change. Use patch_alert instead for promotion, pause, mute/unmute, descriptions, runbooks, annotations, and query_mode. PUT may soft-delete and recreate the rule, so always use the returned replacement ID for subsequent actions. Notification routing belongs to the alert group, not the rule.
